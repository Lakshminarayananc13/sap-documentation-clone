# SAP SD (Sales and Distribution)

SAP Sales and Distribution (SD) is a core SAP module that manages the complete sales process, from receiving customer inquiries to delivering products and generating invoices. It helps organizations efficiently manage sales, shipping, billing, and customer-related activities.

## Overview of SAP SD

SAP SD supports the order-to-cash (O2C) process by managing customer orders, deliveries, billing, and payments.

| Feature                | Description                              |
| ---------------------- | ---------------------------------------- |
| Sales Order Processing | Managing customer orders                 |
| Shipping & Delivery    | Handling product delivery                |
| Billing                | Creating customer invoices               |
| Customer Management    | Managing customer master data            |
| Pricing                | Determining product prices and discounts |

---

## Key Components of SAP SD

### Sales

The Sales component manages customer inquiries, quotations, and sales orders.

| Activity             | Description                                 |
| -------------------- | ------------------------------------------- |
| Inquiry              | Customer request for information            |
| Quotation            | Price offer provided to a customer          |
| Sales Order          | Customer order for products or services     |
| Contract             | Long-term agreement with a customer         |
| Scheduling Agreement | Delivery schedule agreement with a customer |

---

### Shipping

Shipping manages the delivery of products to customers.

| Activity          | Description                    |
| ----------------- | ------------------------------ |
| Delivery Creation | Generate delivery document     |
| Picking           | Select products from inventory |
| Packing           | Pack products for shipment     |
| Goods Issue       | Record shipment of goods       |
| Transportation    | Manage shipment and logistics  |

---

### Billing

Billing creates invoices for delivered goods and services.

| Billing Type         | Description                             |
| -------------------- | --------------------------------------- |
| Invoice              | Standard customer billing document      |
| Credit Memo          | Refund or reduction in customer charges |
| Debit Memo           | Additional charge to customer           |
| Cancellation Invoice | Reversal of an invoice                  |

---

### Customer Master

Customer Master stores all information related to customers.

| Data Type         | Description                             |
| ----------------- | --------------------------------------- |
| General Data      | Customer name, address, contact details |
| Company Code Data | Financial and accounting information    |
| Sales Area Data   | Sales-specific information              |

---

### Pricing

Pricing determines the final amount charged to customers.

| Pricing Element | Description            |
| --------------- | ---------------------- |
| Base Price      | Product selling price  |
| Discount        | Reduction in price     |
| Surcharge       | Additional charge      |
| Tax             | Applicable taxes       |
| Freight         | Transportation charges |

## Order-to-Cash (O2C)

Order-to-Cash (O2C) is the end-to-end process that starts when a customer places an order and ends when payment is received.

### O2C Flow

1. Customer Inquiry
2. Quotation
3. Sales Order
4. Delivery
5. Goods Issue
6. Billing
7. Payment Receipt

The O2C process ensures smooth sales operations, accurate billing, and timely revenue recognition.
